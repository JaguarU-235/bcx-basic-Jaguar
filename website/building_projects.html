building rB+ projects
If you used the rB+ installer, you simply right click the .rbplus file in your project's directory and choose how you would like to build a project.

If you use notepad++ and have it set up correctly, hitting F6 will give you the same four choices.

If you're using the older rB+ setup...

Valid rB+ projects live inside the PROJECTS directory of the rB+ root directory. A project will have its own directory containing a file of the same name as the directory with a .bas extension.

Example:

     rbplus/PROJECTS/doger/doger.bas

ABS projects

To build an rB+ project, navigate to the rB+ root directory and, if sending to Virtual Jaguar, type:

     build [project name]

Example:

     build doger

To build and send to skunkboard, add "sendy"

Example:

     build doger sendy

To build a ROM version of the project, the two examples would look like this:

     build doger ROM

     build doger ROM sendy

The first would send the ROM to VJ, the second would flash the ROM to the skunkboard.


ROM projects

ROM projects are designed to make use of ROM space and obviously require the added ROM command. However, if your ROM file doesn't alter between builds, you can simply create an ABS that accesses the existing flashed ROM without having to write it each time.

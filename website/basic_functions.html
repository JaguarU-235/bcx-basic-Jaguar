rB+ language-specific commands (non-BCX)

LOADCLUT(palette_address,x,y)
     palette_address = address of palette
     x=CLUT destination (0-15)
     y=number of entries to copy (2-256)

    If using the _clut file generated by assets.txt, use LOADCLUT(strptr(graphic_file_clut),x,y)


PLOT(x,y)
     Plot pixel to text/particle layer object at x,y


COLOUR(x)
     x=CLUT entry (0-15)
     Change PLOT colour to entry x from CLUT assigned to text/particle layer.


RLOCATE(x,y)
     Locate cursor at pixel x,y in text/particle layer


RPRINT(hello$)
     Speedier string printing command.


CLS
     Clear text/particle buffer


PEEK x, DPEEK x, LPEEK x
     Read byte, word or longword at address x


POKE x, DPOKE x, LPOKE x
     Write byte, word or longword to address x


RSETLIST(x)
     Change to sprite list x (0-15) defined in rapinit.s


RGETOBJ(x,y)
      Read value from attribute y in sprite x
RSETOBJ(x,y,z)
      Write value z to attribute y in sprite x

      y attributes for RGETOBJ and RSETOBJ:     

       R_sprite_active       R_is_active     R_is_inactive    sprite active or inactive flag
       R_sprite_x                                             x position in 16.16
       R_sprite_y                                             y position in 16.16
       R_sprite_xadd                                          x drift to add in 16.16
       R_sprite_yadd                                          y drift to add in 16.16
       R_sprite_flip         R_is_normal     R_is_flipped     if set, add _width to X and set mirror
       R_sprite_width                                         pixel width of sprite
       R_sprite_height                                        pixel height of sprite
       R_sprite_vbox                                          vertical distance from midpoint for collision detection
       R_sprite_hbox                                          horizontal distance from midpoint for collision detection
       R_sprite_gfxbase                                       address of graphic bitmap data - you can adjust this to animate
                                                                  sprites manually, see next entry
       R_sprite_framesz                                       size of sprite frame in bytes (offset to next frame)
       R_sprite_framedel                                      vertical blanks until next sprite animation frame
       R_sprite_curframe                                      current frame number (or 0 for no frames)
       R_sprite_maxframe                                      maximum frame value (last frame)
       R_sprite_animloop     R_ani_rept      R_ani_once       loop animation forever or just play once
       R_sprite_wrap         R_edge_wrap     R_edge_kill      wrap or make inactive on leaving screen
       R_sprite_timer        #of frames      R_spr_inf        number of frames to stay alive or infinite
       R_sprite_track        R_spr_linear    PTR to x.y       use drift values above or a pointer to a table containing a list
                                                                  of x and y values
       R_sprite_colchk       R_can_hit       R_cant_hit       is sprite to be checked by collision detection routine
       R_sprite_scaled       R_spr_scale     R_spr_unscale    is sprite to be scaled
       R_sprite_scale_x                                       x-scale value. 32 is 1:1
       R_sprite_scale_y                                       y-scale value. 32 is 1:1
       R_sprite_CLUT         #of CLUT        R_no_CLUT        which CLUT to use or none for 16-bit and 24-bit sprites
       R_sprite_animspd                                       frame delay between animations
       R_sprite_bytewid                                       width of one scanline of sprite in bytes
       R_sprite_tracktop                                      loop point for tracking, or -1 for exit
       R_sprite_was_hit                                       set to +ve by the collision routine if sprite hit
       R_sprite_coffx                                         offset from centre of sprite for x collision box location
       R_sprite_coffy                                         offset from centre of sprite for y collision box location
       R_sprite_remhit       R_cd_keep       R_cd_remove      condition to keep or remove when hit in collision detection
       R_sprite_bboxlink     -1                               or pointer to bounding box link data
       R_sprite_hitpoint                                      hit-points of damage to take before removal
       R_sprite_damage                                        hitpoint damage to deal
       R_sprite_flash                                         make sprite blink
       R_sprite_gwidth                                        graphics width in bytes (has to be correctly aligned in memory)


RHIT(sl,sh,tl,th)
     Check collisions between sprites in the range sl to sh with those in the range tl to th. Returns value "1" as global indicator that *something* collided in the range. You can then check each target in turn with RGETOBJ(spriteno,R_sprite_was_hit), reset the R_sprite_was_hit indicator with RSETOBJ(spriteno,R_sprite_was_hit,-1) and manage your collision. Any sprites in the source and target ranges that have the R_sprite_colchk flag set to -1 (cant_hit from sprite definition in rapinit .s) will not be checked.
     The values set in rapinit.s for damage to other sprites and hitpoints for a given sprite can be used to  automatically manage life and end of life conditions of sprites. Any scoring etc. is up to the programmer to implement.

     Example 1:
     check%=RHIT(player,player,enemy1,enemy5)                 ' check a player sprite vs 5 enemies

     IF check=1 THEN                                          ' oops, better luck next time.
          CALL deadplayer
     ENDIF

     Example 2:
     check%=RHIT(bullet,bullet,enemy1,enemy5)                  ' check a single bullet vs 5 enemies

     IF check=1 THEN                                           ' something hit, find out what it was
          FOR i=0 TO 4
               IF RGETOBJ(enemy1+i,R_sprite_colchk)=1 THEN     ' If sprite hi then...
                    RSETOBJ(enemy1+1,R_sprite_colchk,-1)       ' reset hit flag to not hit "-1"
                    score=score+10                             ' manage the player score
               ENDIF
          NEXT i
     ENDIF

       
RUPDALL(x)
     x=type of screen update (0-2)
     0=Update all sprites now
     1=Automatic VBL updating
     2=disable vertical blank updating


VSYNC
     Wait for next VBL.


DELAY(x)
     Wait for x VBLs.


MODPLAY((int)strptr(x))
     x=module number (begins at zero)


SNDPLAY(x,y)
     Play sound x (as defined in rapu235.s) on channel y


MODVOL(x)
     Module volume


SNDKILL(x)
     Kill sounds on channel x


SNDDELTA(x,y)
     x=channel
     y=volume

SNDFREQ(x,y)
     x=channel
     y=volume


SNDPLAYFREQ(x,y,z)
     x=channel
     y=sound
     z=frequency in Hz


SNDVOLRESET(x)
     Reset volume on channel x


RSETMAP(x,y)
     Set tilemap to x,y


RPARTI(strptr(x))
     x=particle array

     Example:

     DIM particles%[511,6]
     particles[0,3]=160<<16                        ' x start
     particles[0,4]=y%<<16                         ' y start
     particles[0,5]=15                             ' particle count
     FOR i=1 TO 15
         particles[i,0]=(i*8)                      ' angle
         particles[i,1]=i/2                        ' speed
         particles[i,2]=0                          ' angular speed
         particles[i,3]=15                         ' initial colour
         particles[i,4]=7                          ' colour decay (per frame)
         particles[i,5]=100                        ' pixel life (in frames)
     NEXT i

     RPARTI(strptr(particles))


GETPAD(x)
     x= joypad (1 or 2)
     Read U235 joypad packet.

     Example:
     pad1=GETPAD(1)
     IF pad1 BAND PAD_UP THEN
          CALL moveup
     ENDIF
